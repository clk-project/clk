<a id="resolution-priority"></a>

## resolution priority

clk resolves names by looking at the local profile first, then the global profile, then aliases and other resolvers. A local custom command always wins over a global alias with the same name, and a global command always wins over an alias.

Several use cases show this in action from different angles:

-   [global workflow, local implementation](global_workflow_local_implementation.md) &ndash; a global `test` alias exists, but when you enter a project that defines a local `test` bash command, the local command wins. The global `test-n-push` workflow automatically picks up the local implementation.
-   [cloud provider CLI wrapper](wrapping_a_cloud_provider_cli.md) &ndash; global parameters set a default AWS profile, but local parameters in a project directory override them. When you leave the project, the global defaults take over again.
-   [alias to root](alias_to_root.md) &ndash; local aliases can be scoped to a project, so they only shadow global names when you are inside that directory.

<a id="cross-project-access"></a>

## cross-project access

When you work on linked but separate projects and get tired of `cd`-ing between them, you can create [aliases that point to the root command](alias_to_root.md) of another project. <a id="introspection-and-debugging"></a>

# introspection and debugging

The `clk describe` command helps you [explore and document your configuration](self_documentation.md), showing what [aliases](#aliases), [parameters](#parameters-and-configuration), commands, and [extensions](#extensions) are available in any profile (global, local, or extension). This is especially useful when [resolution priority](#resolution-priority) makes it unclear which level a setting comes from.

When a command feels slow, use [`--timestamp`, `--debug`, and `--profiling`](spotting_slow_code.md) to progressively narrow down the bottleneck, from high-level timing down to function-level detail.

# plugins

Plugins let you dynamically monkey-patch clk internals (at your own risk).

For most sharing needs, [extensions](#extensions) are a better fit.
